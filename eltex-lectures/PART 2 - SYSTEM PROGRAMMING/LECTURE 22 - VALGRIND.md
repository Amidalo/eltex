Valgrind - динамический анализатор, он проверяет программу в момент ее выполнения. Существуют также статические анализаторы. Valgrind - это не только инструмент для анализа утечек памяти, он может обнаруживать проблемы с примитивами синхронизации, делать базовую профилировку (есть инструмент gprof?).

Для того, чтобы Valgrind прям тыкнул нас носом в строку, где происходит утечка памяти, нужна собрать программу с отладкой.
```
gcc main.c -o main -g
valgrind --leak-check=full ./main
```

Valgrind хорошо работает вместе с gdb, их можно запустить в паре.
```
gcc main.c -o main -g
valgrind --leak-check=full --vgdb=full --vgdb-error=1 ./main
```
Когда происходит ошибка, (action on error) vgdb me, программа зависнет с целью зайти в gdb и подключиться к этой программе.
```
gdb
target remote | vgdb
```

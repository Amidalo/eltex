Препроцессор - это часть компилятора, обрабатывает код программы перед началом компиляции. Препроцессор вставляет содержимое заголовочных файлов, например `#include stdio.h`, прямо в код программы...

#### `define`
Отвечает за определение макроопределений. Когда мы пишем в программе, например `#define N 5`, это означает, что мы создали макрос, то есть макроопределение. Оно имеет имя `N` и значение `5`. После обработки препроцессора (он пробежится по исходному коду нашей программы и найдет все места, где находится наш макрос `N`) это макроопределение заменится на значение `5`.

Если написать следующим образом: `#define max(a, b) ((a > b) ? a : b)`, важно понимать, что макросы не работают как функции. Когда препроцессор найдет место, где используется `max(a, b)`, например `max(c, 15)`, то он просто заменит его на следующее: `((c > 15) ? c : 15)`, то есть `max(c, 15) -> ((c > 15) ? c : 15)`. Важно понимать, что никаких вычислений происходить не будет, так как препроцессор работает на стадии компиляции, то есть мы запустили программу и мы не знаем значение переменной `c`, просто в исходный код подставляется эта тернарная операция, а когда переменная станет известна, мы сможем посчитать то выражение, чтобы получить нужное значение. Стоит также знать, что вместо переменных можно подставить функции, которые что-то возвращают, и сравнить возвращаемые значения.

#### Условная компиляция
В выражениях внутри препроцессинга можно использовать только макроопределения:
```c
#define A 5
#if A == 5
...
int B = 4;
#if B == 4 // Ошибки не будет, компилятор ничего не скажет, просто он не найдет макрос B 
...
#else [elif]
...
#endif

---

#define N
...
#ifdef N // если определено
... // будет выполняться этот кусок кода. Если не определено, то не будет
#endif

---

#define N
...
#ifndef N // если не определено
... // подставится этот кусок кода
#endif
```

#### Заголовочные файлы
Условная компиляция используется здесь для того, чтобы не происходило повторное включение заголовочного файла, ведь это может приводить к ошибкам.
```c
test.c:
#include "my.h"  // здесь макрос _MY_H_ будет определен впервые -> сюда вставится код заголовочного файла
#include "my2.h" // здесь макрос _MY_H_ уже определен не будет -> сюда код заголовочного файла my.h вставляться повторно не будет, только код my2.h
...

my.h
#ifndef _MY_H_ // если макрос не определен
#define _MY_H_ // определить
...
struct test {};
#endif

my2.h
#include "my.h"
```

#### Предопределённые константы
Вот самые распространённые:
```c
__LINE__     // номер строки для отладки
__FILE__     // указывает на текущее имя файла
__FUNCTION__ // имя функции (возможно это макроопределение изменилось)
__DATE__     // текущая дата на момент компиляции
__TIME__     // текущее время на момент компиляции

printf("LINE = %d, FILE = %s, DATE = %s, TIME = %s\n", __LINE__, __FILE__, __DATE__, __TIME__);
```
